## Discovery — gaps in just-shipped behavior

### P0 — silent data loss on serverless
1. **Fire-and-forget post-response work in `/api/inquire`** was using
   `.catch(() => {})` without `after()`. On Vercel, the function
   instance can terminate the moment the response is returned, killing
   in-flight SMS / email / webhook / journey-enroll calls before they
   complete. Symptoms: inquiries log a row but no auto-SMS sent, no
   admin notification email, no webhook fanout.

### P0 — input safety
2. **Public API `?limit=` accepts negative values.** `members` and
   `tournaments` had `Math.min(N || default, cap)` which doesn't floor
   at 1 — a `?limit=-5` request slips through to the driver, either
   erroring or returning 0 rows.

## Shipped
1. Wrapped four post-response side-effects in `/api/inquire` with
   `after()` from `next/server` (Next 16): SMS auto-reply, admin
   notify email, `inquiry.created` webhook fanout, journey enrollment.
2. Floored `?limit` at 1 on `/api/v1/members` and `/api/v1/tournaments`
   (games already had it). Now `Math.max(1, Math.min(N, cap))`.
