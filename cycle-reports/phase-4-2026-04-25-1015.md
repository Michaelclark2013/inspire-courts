# Cycle 1 · Phase 4 (Public Site) · 2026-04-25 10:15

## Scope
Conversion, page speed, SEO (technical + content), schema markup, OG tags, sitemap, internal linking, CTAs, trust signals, mobile.

## Discovery — gaps in current public site

### P0 — SEO rich-result wins (free distribution)
1. **No JSON-LD structured data on sport microsites.** Missing `LocalBusiness` + `SportsActivityLocation` schemas. Google won't show sitelinks, business panel, "Things to do" entries.
2. **FAQ schema not emitted from FAQ accordion.** Each sport page already has 4 Q&A pairs in config — wrapping them in FAQ JSON-LD unlocks Google FAQ rich-result panels.
3. **No JSON-LD on tournament detail pages** — they're the highest-traffic public page during event weekends but missing `SportsEvent` schema.

### P1 — internal linking
4. **Sport microsites don't cross-link to other sports** or to scores/tournaments/schedule. SEO and user-flow leak.
5. **Inquiry confirmation page has no follow-on links** — opportunity to push reader to scores/upcoming events while they wait for the rep call.

### P1 — content + trust
6. **No reviews / testimonials section** anywhere on public site. Trust deficit vs. competitors.
7. **No "as seen in" / partner logos** to anchor credibility.

### P2 — performance
8. Sport microsites should set `revalidate` for ISR (they're statically prerenderable, so this is a no-op but explicit-is-better).
9. No dimensions on hero pattern divs (CLS risk — minor since they're absolutely positioned).

## Chosen tasks

1. **JSON-LD: LocalBusiness + SportsActivityLocation + FAQ on sport microsites.** One `<script type="application/ld+json">` block per sport page, generated from existing config.
2. **Cross-linking footer on sport microsites** — "Other sports at Inspire" section linking to the other 3 sports + scores + tournaments + schedule.
3. **JSON-LD: SportsEvent schema on tournament detail pages.**
4. **Verify robots.txt** is permissive for public, restrictive for admin/portal/api (already done on inspection — confirm in report).

## What was done

### Shipped
1. **JSON-LD on sport microsites** — every sport page (basketball, volleyball, pickleball, futsal) now emits two `<script type="application/ld+json">` tags:
   - `SportsActivityLocation` (LocalBusiness-compatible) with name, description, sport, address, parent org, telephone.
   - `FAQPage` generated from the existing `config.faqs` array — unlocks Google FAQ rich-result panels.
2. **JSON-LD on tournament detail pages** — `SportsEvent` schema with name, description, startDate/endDate, location (Inspire Courts AZ, Gilbert), organizer. Server-rendered so crawlers see it on first paint.
3. **Cross-linking section on sport microsites** — new section above the final CTA with two columns: "Other sports at Inspire" (links to the 3 other sport pages) + "While you're here" (links to /scores, /tournaments, /schedule, /facility). Improves internal-linking SEO + reduces bounce.
4. **Robots.txt audit confirmed** — already correctly disallows `/admin/`, `/portal/`, `/api/`, login routes; allows everything public; sitemap pointer is correct.
