# Cycle 1 · Phase 3 (Admin Panel) · 2026-04-25 10:08

## Scope
Workflow speed, bulk actions, data viz, RBAC, search/filter, keyboard shortcuts, export, undo, optimistic UI, real-time updates.

## Discovery — gaps in current admin panel

### P0 — speed-of-use on the freshest surface
1. **`/admin/inquiries` has no name/phone/email search.** Reps will scan a 200-row list daily — can't type a name to find a returning lead.
2. **`/admin/inquiries` doesn't auto-refresh.** New inquiries don't appear without manual refresh; SLA pipeline must feel live.
3. **No CSV export from `/admin/inquiries`.** Marketing wants to pull weekly lead lists to slice in Sheets.
4. **Inquiry detail drawer doesn't show notes timeline.** The full conversation history is in `inquiry_notes` but the UI never loads `/api/admin/inquiries/[id]` — only the row data.
5. **Status changes do a full reload** instead of optimistic UI. Slow feel; rep clicks "contacted" and waits 800ms.

### P1
6. No bulk actions (mass-mark-contacted, mass-export, mass-reassign).
7. No keyboard shortcuts (j/k navigation, c=contacted, w=won).
8. Sport stats per-source funnel breakdown not visualized.

## Chosen tasks

1. **Live name/phone/email search box** on `/admin/inquiries` — debounced, client-side filter (server already returns up to 200).
2. **Auto-refresh every 30s** on visible tab.
3. **CSV export** — link with same filters appended, server returns text/csv.
4. **Notes timeline in detail drawer** — fetch `/api/admin/inquiries/[id]` on open, render timeline.
5. **Optimistic status update** — patch first, revert on error.

## What was done

### Shipped
1. **Live search** on `/admin/inquiries` — debounced client-side filter across name/email/phone/sports/source. Stats strip auto-reflects filtered count.
2. **Auto-refresh every 30s** when tab visible — uses `visibilitychange` event so we don't burn requests on a hidden tab.
3. **CSV export** — `/api/admin/inquiries?format=csv` returns RFC-4180-compliant CSV with current filters applied. Filename auto-stamped with date.
4. **Notes timeline in detail drawer** — drawer now fetches `/api/admin/inquiries/[id]` on open and renders the chronological note feed (status changes, SLA alerts, SMS sends, manual notes) with kind labels.
5. **Optimistic status updates** — clicking Contacted/Qualifying/Won/Lost mutates the row in UI immediately; on PATCH failure, the previous state is restored. Drawer also reflects the optimistic change.

### Decisions made autonomously
- Search is client-side filter (server returns 200 max already). Cheap, instant, no extra API. If we ever exceed 200 results in a single status×kind filter we'll switch to server-side.
- Auto-refresh interval 30s. Trades freshness for request volume. Faster intervals would surface new leads quicker but burn more cron-ish backend traffic.
- CSV escapes per RFC 4180 (double-quoted, doubled-up internal quotes).
