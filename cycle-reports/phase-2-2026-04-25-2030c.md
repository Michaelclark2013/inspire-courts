**Date:** 2026-04-25 20:30 (round C)

## Gap fixed
`runBillingTick` charged each subscription one at a time via sequential
`await chargeSubscription(...)`. Each Square call is ~500ms; a daily run with
100 due subs approached Vercel's 60s cron timeout. Switched to bounded-parallel
processing — 5 charges in flight at once — so wall-clock drops ~5× while
staying well under Square's per-app rate limit. Each charge owns disjoint DB
rows, so there's no shared-state contention.
