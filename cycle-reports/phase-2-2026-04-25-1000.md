# Cycle 1 · Phase 2 (Functionality) · 2026-04-25 10:00

## Scope
Feature completeness, edge cases, error handling at boundaries, performance, integrations, validation, forms.

## Discovery — gaps in what's currently shipped

### P0 — public surfaces hardening
1. **`/api/inquire` has no honeypot or anti-spam.** Public endpoint, will get bot traffic. Rate limit alone is insufficient.
2. **Email + phone validation is loose.** API accepts any string; bad data flows downstream into SMS/email send attempts that will silently fail.
3. **No SLA breach alerting.** An inquiry can sit "new" past 30 min with no rep notified. The whole 30-min SLA promise leaks if nobody actually pings the rep.

### P0 — integrations
4. **New inquiries don't auto-enroll into a "new lead" journey.** We built the journey engine but inquiries skip it. Manual follow-up is the only signal.

### P1 — forms / UX
5. **InquiryForm has no client-side phone format hint or auto-format.** US phone numbers entered with mixed formatting are accepted but harder to dial.
6. **Admin /inquiries list has no real-time refresh.** Reps have to manually refresh during shift.

### P1 — performance
7. **Sport microsites are dynamic-rendered** (`ƒ` in build output) because they share a layout that may opt out of static. Should be statically prerenderable for SEO.

### P2 — micro
8. Search by name/phone/email on /admin/inquiries (only kind+status filter exists).
9. Bulk reassign on /admin/inquiries.

## Chosen tasks for iteration

1. **Honeypot field on /api/inquire** — invisible `website` field; if filled, silently 200 and drop the row.
2. **Strict email + E.164 phone validation server-side** — reject early with a clear error.
3. **SLA breach cron** — `/api/admin/cron/inquiry-sla-tick` every 5 min finds inquiries `status='new'` with `slaDueAt < now`, posts a single notification (uses existing notify channel), stamps a `slaNotifiedAt` so we don't spam.
4. **Auto-enroll new inquiries into a "new_lead" journey** if one exists and is active.

Skipping for next phase 2 iteration: phone auto-format (UX), realtime admin refresh, search filter on admin list, sport-pages static-rendering verification.

## What was done

### Shipped
1. **Honeypot anti-spam** on `/api/inquire` — hidden `website` field; if filled, returns fake-success and silently drops. Client form includes the hidden input with `tabIndex={-1}` and absolute-positioned offscreen.
2. **Server-side validation hardening** on `/api/inquire`:
   - Email regex check (rejects malformed before DB insert + downstream send).
   - Phone normalized to E.164 (`+1XXXXXXXXXX`); reject if not 10 or 11 digits. Twilio + downstream stay clean.
   - Email lowercased pre-insert for matching consistency.
3. **SLA breach cron** at `/api/admin/cron/inquiry-sla-tick` — runs every 5 min via Vercel cron. Finds `status='new'` inquiries with `firstTouchAt IS NULL` and `slaDueAt < now` that haven't already been alerted (uses `inquiry_notes` with `kind='sla_alert_sent'` as the dedupe). Emails on-call rep + drops a note row so subsequent ticks skip.
4. **Auto-enroll into "new_lead" journey** — when a new inquiry's email or phone matches an existing member, auto-enrolls them into any active journey with `trigger='member_created'`. No-op when no member match (admin can manually enroll after creating the member record).
