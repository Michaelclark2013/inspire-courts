## Gaps found this iteration

### P0 — fix this iteration
1. **Inbox `send` swallows POST errors** — the SMS send button always reset `reply` and reloaded threads, even on a 4xx/5xx. User had no idea the message wasn't delivered. Added an inline error banner + `aria-busy` + retry-friendly UX.
2. **Inbox send button under 44×44** — `py-2` on a square icon button = 32px tall. Bumped to `min-h-[44px] min-w-[44px]` and added `focus-visible` + `aria-label`.
3. **ConfirmModal backdrop missing `cursor-pointer`** — the backdrop is click-to-dismiss but has no cursor hint. Added.
4. **Plain "Loading…" text on three admin lists** — announcements, permissions index, permission templates. Replaced with `<SkeletonRows>` so the page doesn't collapse to a single short line during the fetch.

### P1 — next iteration
- Audit log / inbox don't surface fetch errors to user — they swallow them and render an "empty" state that's misleading.
- Badge `neutral` variant uses `text-text-secondary` which can fail WCAG AA on off-white card backgrounds.
- ExportBar buttons are still `py-2` (under 44px).

### P2 — someday
