## Gap fixed
Square webhook handler treated a missing `SQUARE_WEBHOOK_SIGNATURE_KEY`
as "skip verification" — it would log a warning and then continue
processing the request. The downstream code flips
`tournamentRegistrations.paymentStatus = "paid"` and adds the team
to the bracket. If a partial Square config ever shipped to production
(access token set but webhook signature key forgotten), anyone on the
internet could POST a crafted JSON body to /api/webhooks/square and
mark arbitrary registrations as paid.

Today the practical risk is zero — Square isn't configured at all, so
no order IDs exist to match. But the moment Mike sets the access token
without also setting the webhook signature key, this becomes a real
vulnerability.

Fix: in production (`NODE_ENV === "production"`), refuse to process
unsigned webhooks (return 503). In dev/sandbox the loose behavior is
preserved for local testing, but the log moves to `logger.error`-style
loudness so the missing-key state is obvious.
