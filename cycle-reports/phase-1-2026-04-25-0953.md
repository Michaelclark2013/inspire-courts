## Gaps identified (ranked)

### P0 — accessibility (WCAG 2.1 AA)
1. **No skip-to-main-content link.** Keyboard users tab through the entire header on every page. WCAG 2.4.1 requirement.
2. **Sport program tile links lack visible focus rings.** Custom hover styling (`hover:border-red/40`) hides default focus indicator. Keyboard users can't see where they are.
3. **InquiryForm `<input>` focus rings inconsistent** with site brand — currently use native browser styles in some places.
4. **`<details>` FAQ accordion has no animated transition** — works but feels janky on toggle.

### P0 — motion safety
5. **No `prefers-reduced-motion` global support.** Animated elements (loading spinners, hover transitions, ping pulse on LIVE chip) play even when users have OS-level "reduce motion" set. WCAG 2.3.3.

### P1 — visual polish
6. **Sport microsite hero is text-only**, no imagery / pattern / depth. Looks flat next to competitors.
7. **InquiryForm success state has no animation** — abrupt swap from form to success card.
8. **Sport stats strip cards lack visual hierarchy** — big number doesn't pop.

### P2 — micro-polish
9. Sport program tile hover state could use subtle scale.
10. Mobile sport hero text size at `text-7xl` may overflow on very small screens (iPhone SE).

## Chosen tasks for iteration 1

I'll ship the four highest-leverage fixes (P0 + small visual polish):

1. **Add skip-to-main-content link** in root layout
2. **Global `prefers-reduced-motion` CSS** in globals.css — disable transitions/animations for users who request it
3. **Focus-visible rings** on sport program tiles + sport hero CTAs + sport FAQ summary + InquiryForm fields
4. **Sport hero visual depth** — subtle dot-grid pattern + animated gradient blob (already partial, enhance)
5. **InquiryForm success animation** — fade-in scale on success card
