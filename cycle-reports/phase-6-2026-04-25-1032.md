# Cycle 1 · Phase 6 (Security & Privacy) · 2026-04-25 10:32

## Scope
OWASP top 10, secrets, env audit, npm audit, CSP/HSTS, auth hardening, rate limits, input validation, XSS/CSRF, PII handling, GDPR/CCPA, audit logging.

## Discovery — current state

### Strong already
- **Security headers solid** in `next.config.ts`: CSP w/ explicit allowlists, HSTS w/ 2-year max-age + preload, X-Frame-Options DENY, X-Content-Type-Options nosniff, Referrer-Policy strict-origin-when-cross-origin.
- **Audit log** infrastructure exists; admin routes use `recordAudit()`.
- **Rate limit** library exists; applied on `/api/inquire`, `/api/portal/clock`, etc.
- **API keys** sha256-hashed in storage, prefix-only display (`/admin/integrations`).
- **Inquiry form**: honeypot + email/phone validation + length caps (just shipped phase 2).
- **Cookie consent** banner present; `Permissions-Policy` set.

### Gaps found

#### P0
1. **`Permissions-Policy: geolocation=()`** in next.config.ts BLOCKS geolocation site-wide. We just shipped geofenced clock-in (phase 1 of cycle, before this rotation) and the portal/staff/clock page calls `navigator.geolocation`. Currently silently failing in browsers that honor the policy. Must allow `(self)`.
2. **Twilio webhook lacks signature verification.** `/api/webhooks/twilio` accepts any POST and processes it as a real inbound SMS. An attacker could spoof STOP messages to opt-out anyone, or fake replies to halt journey enrollments. Must verify `X-Twilio-Signature` against `TWILIO_AUTH_TOKEN`.
3. **Public API `/api/v1/*` has no rate limit.** A leaked or stolen key could flood the DB. Rate limit per-key.

#### P1
4. **11 moderate npm vulns** — all in transitive deps (esbuild via drizzle-kit, uuid via next-auth/svix/resend, postcss via next, nodemailer minor). All require `--force` upgrades that bump majors. Risky; deferred to a dedicated dependency rotation.
5. **No CSRF token** on admin POST/PATCH routes — relies on same-site session cookie + role check, which is good baseline but adds defense-in-depth value to add explicit CSRF token to mutating admin routes.

#### P2
6. CSP `connect-src` doesn't include `js.squareup.com` — will need adding when Square Web Payments SDK lands client-side for billing card capture.

## Chosen tasks

1. **Fix Permissions-Policy** — `geolocation=(self)` so clock-in geofence works.
2. **Twilio webhook signature verification** — use `twilio` SDK's `validateRequest`.
3. **Per-API-key rate limit** on `/api/v1/*` — 60 req/min per key.
4. **Document deferred npm vulns** in this report.

## What was done

### Shipped
1. **Permissions-Policy fixed** in `next.config.ts` — `geolocation=(self)` instead of `()`. Unblocks geofenced clock-in across all browsers that honor the policy.
2. **Twilio webhook signature verification** — `/api/webhooks/twilio` now uses `twilio.validateRequest` against `TWILIO_AUTH_TOKEN`. Reads body as text, parses to URLSearchParams, reconstructs full URL from `x-forwarded-proto + host`. Returns 403 on invalid signature, 401 on missing token. Closes the spoofed-STOP/spoofed-reply attack vector.
3. **Per-API-key rate limiting** — new `checkApiKeyRateLimit` in `src/lib/api-auth.ts`, 60 req/min per key, in-memory bucket. Wired into all three `/api/v1/*` endpoints (members, games, tournaments). Returns 429 with `Retry-After` header on breach.
