# Cycle 2 · Phase 5 (Marketing) · 2026-04-25 23:15

## Discovery
1. UTM attribution was URL-only — visitor lands /basketball?utm=
   instagram, browses 5 min to /inquire/training, attribution lost
   because the form reads window.location.search at submit time.
2. No public iCal feed for tournaments — TDs and visiting coaches
   have to manually check /tournaments. Big loyalty + free-distribution
   miss.

## Shipped
1. **First-touch attribution persistence** (`src/lib/attribution.ts`):
   - localStorage with 30-day TTL
   - First-touch wins (multi-day consideration cycle for memberships
     means the campaign that earned the lead matters most)
   - getAttribution() prefers URL utm, falls back to storage
2. **AttributionTracker component** mounted in root layout via dynamic
   import + Suspense (uses useSearchParams). Fires capture on every
   client-side navigation.
3. **InquiryForm** now uses getAttribution() instead of inline URL parse.
4. **Public iCal feed** at `/api/calendar/tournaments.ics`:
   - VCALENDAR with VEVENT per upcoming published/active tournament
   - All-day events (DTSTART;VALUE=DATE) with exclusive DTEND per RFC
   - Stable UID `tournament-{id}@inspirecourtsaz.com` so updates
     replace rather than duplicate in subscribers' calendars
   - 1-hour CDN cache + stale-while-revalidate
   - RFC 5545 escaping + 75-octet line folding
