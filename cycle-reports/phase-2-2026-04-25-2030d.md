## Gap fixed
Tournament division capacity check
(`src/app/api/tournaments/[id]/register/route.ts:113`) only counted
approved-or-paid registrations. Pending-but-not-paid teams (i.e.
mid-Square-checkout) didn't fill a slot. Failure mode: N concurrent
/register POSTs into a near-full division all create pending rows
simultaneously, none count yet, each generates a Square checkout, all
get paid → division ends up overbooked.

Fix: count recent pending registrations (created within the last 24
hours) in addition to approved/paid. Older pending rows are treated
as abandoned and still excluded so they don't permanently lock slots.
